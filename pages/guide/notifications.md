# 消息推送 (PushPlus)

PicHub 集成了 [PushPlus](https://www.pushplus.plus/) 推送服务，可将系统关键事件实时推送到你的微信。

## 支持的通知事件

| 事件 | 推送内容 | 模板 |
|---|---|---|
| 支付成功 | 金额、支付方式、订单号、商品名 | pay |
| 新用户注册 | 用户名、邮箱、注册时间、IP | html |
| 每日统计 | 新用户数、上传量、收入、存储用量 | markdown |
| 举报待处理 | 待处理举报数量、最新举报时间 | html |
| 系统错误 | 错误类型、错误信息、发生时间 | html |
| 新工单创建 | 工单主题、提交者、优先级、内容预览 | html |
| 工单回复 | 工单主题、回复者、回复内容预览 | html |

## 配置步骤

### 1. 获取 PushPlus Token

1. 访问 [pushplus.plus](https://www.pushplus.plus/) 并用微信扫码登录
2. 在个人中心复制你的 **Token**

### 2. 在后台配置

登录 Filament 管理面板 → **系统设置** → **消息推送**：

| 设置项 | 说明 |
|---|---|
| PushPlus Token | 填入上一步获取的 Token |
| 启用推送 | 全局开关 |
| 各事件开关 | 可单独控制每种事件是否推送 |

### 3. 发送测试

配置完成后，点击「发送测试通知」按钮，确认微信能收到推送。

## 推送模板

PushPlus 支持多种消息模板，PicHub 根据事件类型自动选择最合适的模板：

| 模板类型 | 适用场景 | 效果 |
|---|---|---|
| `html` | 大多数通知 | 富文本，支持颜色和排版 |
| `markdown` | 每日统计 | 简洁的结构化内容 |
| `pay` | 支付通知 | 专门的支付卡片样式 |

## 推送频率控制

为避免过多的推送干扰，PicHub 对频繁事件做了合理的聚合：

- **举报通知**：聚合为"当前有 N 条待处理"，而非每条单独推送
- **系统错误**：相同错误类型在短时间内只推送一次
- **每日统计**：每天定时推送一次，通过 `php artisan schedule:run` 触发

## 推送日志

所有推送记录都保存在数据库中，可在 Filament 管理面板 → **系统设置** → **推送日志** 中查看：

- 推送时间
- 推送类型
- 推送状态（成功/失败）
- 错误信息（如有）

> [!TIP]
> 如果推送失败，常见原因是 Token 过期或 PushPlus 服务临时不可用。检查 Token 是否正确、PushPlus 账户是否正常。
